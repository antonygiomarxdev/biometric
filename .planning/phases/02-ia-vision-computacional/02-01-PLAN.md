  <name>Task 2: Implement Training Loop for U-Net &amp; Alternative Architectures</name>
  <files>scripts/spike_enhancement.py</files>
  <action>
    Extend `scripts/spike_enhancement.py` with:

    **U-Net baseline (per D-03):**
    - Use `segmentation-models-pytorch` to create a U-Net with `mobilenet_v2` encoder (lightweight, good for 8GB VRAM), `in_channels=1`, `classes=1`.
    - Implement training loop with perceptual loss (use kornia's SSIM + L1 combination — no hand-rolled loss function per RESEARCH.md Don't Hand-Roll).
    - Use AdamW optimizer, cosine annealing LR scheduler.
    - Implement validation metrics: PSNR, SSIM (via kornia.losses.SSIM or torchvision), and a simple "minutiae recovery rate" that counts detected minutiae before/after enhancement using the existing `SkeletonMinutiaeExtractor`.

    **Alternative baseline (simple CNN denoiser):**
    - Implement a lightweight 6-layer CNN autoencoder as comparison.
    - Same loss function, optimizer, and evaluation protocol.

    **Comparison logic:**
    - Train both models for the same number of epochs (configurable, default 50).
    - Record: training loss, validation PSNR, validation SSIM, minutiae recovery rate.
    - Print comparative table at end.

    Both architectures must use ONNX export at the end of training:
    - `torch.onnx.export(..., opset_version=18)` per RESEARCH.md Pitfall 3.
    - Validate the exported ONNX model loads with `onnxruntime.InferenceSession`.
    - Save to `data/models/spike_unet.onnx` and `data/models/spike_cnn.onnx`.

    Log training progress with tqdm for user visibility.
    Keep all code comments in English per D-06.
