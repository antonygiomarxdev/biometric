<objective>
Train a binary CNN classifier (FineNet-style) that distinguishes real minutiae from noise, export it to ONNX, and verify it loads and runs on the RTX 4070 GPU.

Purpose:
The original plan assumed a pre-trained ONNX model. PATH D triggered because none exists publicly. Per user decision (Option A, 2026-06-15), we build our own FineNet-style classifier. The architecture is a small CNN that takes a 160x160 grayscale patch (centered on a candidate minutia) and outputs a probability of being a true minutia. Training data comes from SOCOFing Real: positive patches centered on `ground_truth_count.py` minutiae, negative patches centered on random non-minutia points. After training, the model is exported to ONNX and saved to `data/models/extract.onnx`. The smoke test validates it loads with CUDA and infers in <50ms.

Output: Trained ONNX model, training script, dataset generation script, smoke test, MODEL_CARD.md update.
</objective>

  <action>
Create `apps/backend/scripts/generate_patch_dataset.py` (no FastAPI deps):

1. Iterate over 200 SOCOFing Real images (subset of 6000 — full set would create class imbalance)
2. For each image:
   a. Load grayscale with cv2
   b. Resize to 256x256 if smaller (most SOCOFing images are 96x103 — too small for 160x160 patches; resize with cv2.INTER_CUBIC)
   c. Call ground_truth_count logic from `apps/backend/scripts/ground_truth_count.py` to get true minutia (x, y) positions
   d. For each true minutia, extract 160x160 patch centered on it (pad with reflection if near border) → label 1 (positive)
   e. For each image, sample 3 random non-minutia points (not within 16px of any true minutia) → 160x160 patch → label 0 (negative)
3. Save patches as `apps/backend/data/training/patches.npz` (N, 160, 160) uint8
4. Save labels as `apps/backend/data/training/labels.npz` (N,) uint8
5. Print class balance: positives, negatives, total

Target: ~2000-3000 positive patches, ~600-900 negative patches (3:1 ratio of pos:neg, since real-world ratio is closer to that for the CPU extractor output).

CRITICAL: Use train/test split BEFORE saving. Save indices for `train_indices` and `test_indices` (80/20 split, random with seed=42) so the training script can use the same split.

Also save a `metadata.json` with: total count, positive count, negative count, image size used, random seed.

**IMPORTANT:** Do NOT include the test images in the training set. The test set is what we use to measure real accuracy.

<task type="auto">
  <name>Train CNN classifier (PyTorch)</name>
  <files>apps/backend/scripts/train_patch_classifier.py</files>
  <action>
Create `apps/backend/scripts/train_patch_classifier.py`:

Architecture (simple but effective for 160x160 binary):
```
Input: (1, 160, 160)
Conv2d(1, 32, 3, padding=1) + BatchNorm + ReLU + MaxPool(2)    # (32, 80, 80)
Conv2d(32, 64, 3, padding=1) + BatchNorm + ReLU + MaxPool(2)   # (64, 40, 40)
Conv2d(64, 128, 3, padding=1) + BatchNorm + ReLU + MaxPool(2)  # (128, 20, 20)
Conv2d(128, 256, 3, padding=1) + BatchNorm + ReLU + MaxPool(2) # (256, 10, 10)
AdaptiveAvgPool2d(1) → Flatten → Linear(256, 1) → Sigmoid
```

Hyperparameters:
- Loss: BCEWithLogitsLoss (use logits, not sigmoid output)
- Optimizer: Adam, lr=1e-3
- Batch size: 32
- Epochs: 30 (with early stopping patience=5 on val loss)
- Augmentation: random horizontal/vertical flip (50%), random rotation ±15° (for orientation invariance), random brightness ±10%

Training loop:
1. Load patches.npz and labels.npz
2. Use train_indices for training, test_indices for validation
3. Create DataLoader with augmentation for train, no augmentation for val
4. Train with early stopping
5. Save best model (lowest val loss) to `data/models/extract.pt`
6. Print final test accuracy, precision, recall, F1
7. Target: test accuracy >=85%, F1 >=85%

**ONNX export step** (after training):
1. Load best checkpoint
2. Set model to eval mode
3. Create dummy input: torch.randn(1, 1, 160, 160)
4. torch.onnx.export(model, dummy, "data/models/extract.onnx",
                       input_names=["input"], output_names=["output"],
                       dynamic_axes={"input": {0: "batch"}, "output": {0: "batch"}},
                       opset_version=13)
5. Verify exported model with onnxruntime

**CRITICAL:**
- Do NOT use sklearn or any heavy framework — only torch, numpy, onnx
- Strict types: no `Any` (we'll check with pyright at the end)
- Save training log: `data/training/train_log.json` with per-epoch loss/acc
  </action>
  <verify>

    Test accuracy >= 85% printed
