  <action>
Create `apps/backend/scripts/eval_kpi.py` (runnable as `python -m apps.backend.scripts.eval_kpi`):

**Sample selection:**
- 12 images from `static/SOCOFing/Real/` (filename pattern `*_M_*.BMP` to get one finger per subject)
- 12 images from `static/SOCOFing/Altered/Altered-Easy/`
- 12 images from `static/SOCOFing/Altered/Altered-Medium/`
- 12 images from `static/SOCOFing/Altered/Altered-Hard/`
- Use `random.seed(42)` for reproducibility
- Total: 48 images

**For each image:**
1. Load image (grayscale)
2. Run `FingerprintService(use_ai_extractor=True).process_image(image)` — measure wall time
3. Extract `len(candidates)` from output
4. Get ground truth count from `ground_truth_count.py` (CN on binarized+thinned image)
5. Compute: True Positives = candidates within 8px of any ground truth minutia; FP = candidates >8px from any ground truth; FN = ground truth minutiae with no candidate within 8px

**Per-category metrics:**
- Recall = TP / (TP + FN)
- Precision = TP / (TP + FP)
- F1 = 2 * P * R / (P + R)
- FPR = FP / total_extracted
- Latency = mean + p95

**Output:**
- `test_results/kpi_eval/report.json` — raw numbers, machine-readable
- `test_results/kpi_eval/REPORT.md` — markdown with per-category table, pass/fail indicators next to each KPI
- Print summary to stdout

**Important:**
- Use a 8px matching radius (fingerprint minutiae in 500ppi are 8-12 px apart)
- The ground truth from `ground_truth_count.py` is itself a CN-based estimate, not a hand-labeled ground truth — note this limitation in the report
- The FPR for "Altered-Hard" is the KEY metric (this is where the CPU pipeline had 211 FP)
