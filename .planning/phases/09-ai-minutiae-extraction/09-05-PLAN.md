```markdown
## AI Minutiae Extraction (Phase 09)

The pipeline supports GPU-accelerated minutiae extraction via ONNX. See
[`data/models/MODEL_CARD.md`](data/models/MODEL_CARD.md) for the model source,
license, and acquisition instructions.

### Environment variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `USE_AI_EXTRACTOR` | `false` | Use AI extractor (requires model file) |
| `AI_USE_GPU` | `true` | Try CUDA first, fall back to CPU |
| `AI_MODEL_DIR` | `data/models/` | Directory containing `extract.onnx` |

### GPU deployment

```bash
docker compose -f docker-compose.gpu.yml up
```

The first request will trigger warmup. The startup log shows which provider
is in use. To verify the deployment:

```bash
python -m apps.backend.scripts.verify_ai_deployment
```

### CPU fallback

If the model file is missing or CUDA is unavailable, the pipeline
automatically falls back to the CPU extractor. To force CPU:

```bash
export USE_AI_EXTRACTOR=false
```

### KPI results

See [`test_results/kpi_eval/REPORT.md`](test_results/kpi_eval/REPORT.md)
for the absolute KPI evaluation (Recall, FPR, latency) on the SOCOFing
benchmark.
```
