<objective>
Wire the trained ONNX patch classifier from Plan 01 into the fingerprint pipeline as a post-hook that runs AFTER the CPU extractor and removes candidates that are noise.

Purpose:
Plan 01 produced a trained ONNX classifier. This plan makes it usable through the existing `IPipelineStep` interface, configured to run as a post-hook (not an extractor). The CPU `SkeletonMinutiaeExtractor` still does the candidate generation (~200 noisy candidates); the `OnnxPatchFilter` then runs ONNX inference on a 160x160 patch around each candidate and removes the ones that score below the confidence threshold. The result: same number of candidates, but most noise is gone. This is the FineNet pattern from the MinutiaeNet paper, adapted to our uniform pipeline.

Output: `OnnxPatchFilter` class, config flag, dual-path test coverage, integration with `FingerprintService`.
</objective>

Create `apps/backend/src/processing/onnx_patch_filter.py`:

```python
class OnnxPatchFilter:
    """IPipelineStep post-hook that filters noise candidates via ONNX classifier.

    For each candidate in ctx.candidates, extracts a 160x160 patch centered
    on the candidate, runs ONNX inference, and removes candidates whose
    confidence is below the threshold.

    The CPU SkeletonMinutiaeExtractor typically produces ~200 candidates with
    significant noise. This filter is expected to reduce that to ~50-80
    real minutiae per fingerprint.
    """

    def __init__(
        self,
        model_manager: ModelManager,
        patch_size: int = 160,
        confidence_threshold: float = 0.5,
    ) -> None:
        self.model_manager = model_manager
        self.patch_size = patch_size
        self.confidence_threshold = confidence_threshold

    def process(self, ctx: PipelineContext) -> None:
        # 1. Validate ctx.enhanced_image exists
        if ctx.enhanced_image is None:
            return  # No image to patch from; skip silently

        if not ctx.candidates:
            return  # No candidates to filter

        # 2. Extract 160x160 patches
        patches = self._extract_patches(ctx.enhanced_image, ctx.candidates)

        # 3. Run ONNX inference
        confidences = self.model_manager.run_classification(patches)

        # 4. Filter candidates
        kept = [
            c for c, conf in zip(ctx.candidates, confidences.flatten())
            if conf >= self.confidence_threshold
        ]
        ctx.candidates = kept
```

`_extract_patches(image, candidates)`:
- For each (x, y) in candidates:
  - Extract window [x-80:x+80, y-80:y+80] (with reflection padding if at border)
  - Normalize to [0, 1] float32
- Stack into (N, 1, 160, 160) array

Use `np.pad` with mode='reflect' for border handling.

Strict types. No Any. No type: ignore.

```python
def _default_post_hooks(self, settings: Settings) -> list[IPipelineStep]:
    hooks: list[IPipelineStep] = [QualityFilter(), BorderMaskCleaner(), ...]
    if settings.use_ai_patch_filter:
        try:
            from src.ai.model_manager import ModelManager
            from src.ai.config import AiConfig
            from src.processing.onnx_patch_filter import OnnxPatchFilter

            manager = ModelManager(AiConfig())
            hooks.append(OnnxPatchFilter(
                manager,
                confidence_threshold=settings.ai_patch_confidence_threshold,
            ))
            logger.info("AI patch filter registered (provider=%s)", AiConfig().provider)
        except FileNotFoundError as e:
            logger.warning("AI patch filter requested but model missing: %s. CPU-only.", e)
        except Exception as e:
            logger.warning("AI patch filter failed to initialise: %s. CPU-only.", e)
    return hooks
```

CRITICAL ordering: the AI patch filter must run AFTER the CPU extractor and BEFORE the post-hooks that do final cleanup (e.g. `QualityFilter`, `LowConfidenceFilter`). This is because:
- `EnsembleFusionFilter` (if used) needs all candidates before AI filtering
- `LowConfidenceFilter` works on the AI score, so it must run AFTER AI
- `BorderMaskCleaner` should run AFTER AI to avoid wasting inference on border noise

The simplest correct ordering:
1. [CPU extractors]
2. [EnsembleFusionFilter] (if enabled)
3. **OnnxPatchFilter** (NEW — runs ONNX inference per candidate)
4. [QualityFilter, BorderMaskCleaner, etc.]

Lazy import inside the function (avoid forcing onnxruntime import on CPU-only deployments).
