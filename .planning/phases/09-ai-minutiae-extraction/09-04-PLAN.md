<task type="auto">
  <name>FastAPI lifespan startup hook</name>
  <files>apps/backend/src/main.py</files>
  <action>
In `apps/backend/src/main.py`, modify the FastAPI lifespan context manager:
- On startup: instantiate `ModelManager(AiConfig())`, call `manager.warmup()`, log the provider used ("CUDA" or "CPU")
- If `use_ai_extractor` is True and warmup failed for "extract", log a WARNING that the AI path will fall back to CPU
- Store the manager in `app.state.model_manager` so request handlers can access it without re-instantiating

Example:
```python
@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncIterator[None]:
    manager = ModelManager(AiConfig())
    results = manager.warmup()
    logger.info("AI provider: %s, warmup: %s", AiConfig().provider, results)
    app.state.model_manager = manager
    yield
    manager.unload_all()
```

Update `FingerprintService` to read from `app.state.model_manager` if available, falling back to its own instantiation. This is the cleanup that prevents per-request session creation.
  </action>
