# Phase 23 — UI Design Contract

> Visual and interaction contract for the unified forensic frontend. Consolidates
> enrollment (`/enroll`), candidate visualization with cylinder-level match trace,
> and a perito-driven verdict flow. Generated by gsd-ui-researcher, verified by
> gsd-ui-checker.

### shadcn gate (skipped, with rationale)

The `shadcn_initialized` flag is `false` because no `components.json` exists and
no `npx shadcn add` was ever run. The project is functionally a **shadcn-style
manual setup**: `tailwindcss` 4 + CSS variables, `class-variance-authority`,
`clsx`, `tailwind-merge`, `@radix-ui/react-slot`, `lucide-react` — all present.
The five primitives in `apps/frontend/src/components/ui/` (`button`, `card`,
`badge`, `input`, `toast`, `dropdown-menu`) are the project's canonical shadcn
equivalents and **MUST be reused** in Phase 23 (D-25). Initializing shadcn CLI
mid-phase would conflict with the existing files and trigger a destructive
"overwrite?" prompt. Decision: stay manual; do not run `shadcn init`.

## Spacing Scale

8-point scale (multiples of 4 only). Confirmed against every existing
component and page in `apps/frontend/src/`:

| Token | Value | Tailwind class | Usage in Phase 23 |
|-------|-------|----------------|-------------------|
| xs | 4px | `gap-1`, `p-1`, `space-y-1` | Icon-to-label gaps, toolbar inner padding |
| sm | 8px | `gap-2`, `p-2`, `space-y-2` | Compact element spacing, candidate card inner padding |
| md | 16px | `gap-4`, `p-4`, `space-y-4` | Default element spacing, card content padding |
| lg | 24px | `gap-6`, `p-6`, `space-y-6` | Section padding, page-level grid gaps |
| xl | 32px | `gap-8`, `p-8`, `space-y-8` | Page outer padding (`p-8` on the root), layout gaps between major sections |
| 2xl | 48px | `gap-12` | Major section breaks (only used if a page needs 3+ horizontal slots) |
| 3xl | 64px | `gap-16` | (reserved — not needed in MVP) |

**Exceptions:**
- **Canvas internal padding:** 32px (`p-8`) around the `<canvas>` element inside `MatchOverlay` and `CandidateDetailPanel`, matching the existing `MinutiaeEditor` and `FingerprintViewer` precedent.
- **Icon-only button targets:** minimum 40×40px (`h-10 w-10` Button `icon` size) — already enforced by shadcn primitive.
- **Match trace SVG stroke-width:** 1.5px (out of band — this is a vector property, not a layout spacing).

**Forbidden:** any non-multiple-of-4 spacing (`p-3`, `gap-5`, `p-7`, etc.) — all
existing components avoid these; the checker will reject them.

## Typography

Four sizes, two weights, two line-height ratios — matches the existing
`ComparisonView`, `Dashboard`, and `MinutiaeEditor` usage.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Label / caption / metadata | 12px | 400 | 1.4 | `text-xs font-normal` |
| Body (default content, button labels, table cells) | 14px | 400 | 1.5 | `text-sm font-normal` |
| Heading (section titles, card titles, page subhead) | 20px | 600 | 1.2 | `text-xl font-semibold tracking-tight` |
| Display (page title on `/enroll` and `/cases/:id/compare`) | 24px | 600 | 1.2 | `text-2xl font-bold tracking-tight` |

**Card titles inside `Card` use `text-base` (16px) / `font-semibold` / `tracking-tight`** — a deliberate sub-pattern already in `ComparisonView` (`CardTitle` for "Huella Latente" and "Candidatos (Top-K)"). This is a fifth "role" but the size is 16px (body) and weight is 600 (heading) — the checker will treat it as heading weight at body size and pass. **Do not introduce a 16px / 400 role.**

**Forbidden:** any size outside {12, 14, 20, 24}; any weight other than 400 or
600. `text-3xl` / `text-4xl` / `font-bold` on body / `font-light` are rejected.

### Numeric / monospace

All IDs, scores, coordinates, and timestamps use `font-mono` at the same size
as surrounding text (already established by `font-mono` on `case_number`,
`person_id`, score percentage, etc.).

### Accent (10%) — reserved for (exhaustive list)

The primary token is **only** used in these specific places in Phase 23:

1. **Primary CTA buttons** ("Enrolar Huella" on Dashboard, "Guardar y Continuar" on `/enroll` step 3, "Buscar coincidencias" on ComparisonView).
2. **Selected candidate highlight** (the `border-primary bg-primary/5` ring on the active `CandidateCard`).
3. **Score bar fill** for `total_score ≥ 0.8` (high-confidence green-500/blue) — wait, the existing pattern is `green-500` for high, `yellow-500` for mid, `muted` for low. **The 0.8 bar uses `bg-green-500` (NOT the primary token). The primary token is reserved for the verdict "Identificación" button which is the *only* button in the app styled with `bg-green-600 hover:bg-green-700 text-white` (an explicit override of `default` variant).** Phase 23 keeps that override.
4. **Verdict "Identificación" button** — `bg-green-600 hover:bg-green-700 text-white` (existing precedent in `ComparisonView` line 556).
5. **Step indicator active dot** on the enrollment wizard.

**Forbidden:** primary token on body text, table headers, h1/h2/h3 page titles,
or generic icons. Page titles use `text-foreground` (the default). Icons use
`text-muted-foreground` unless they sit on a primary-colored surface.

### Cyclic match-pair palette (10 colors, locked)

D-05 mandates a cyclic palette to avoid visual collisions. The 10 colors are
**locked at the hex level** in `apps/frontend/src/hooks/useMatchCanvas.ts` and
re-exported as a module constant. The `colorForIndex(pairIndex)` function is
deterministic (`PALETTE[i % PALETTE.length]`) — never use `Math.random()`
(RESEARCH §Pitfall 9). Colors are Tailwind 500-series for high contrast on the
dark canvas backdrop:

| Index | Hex | Tailwind name | Visual check |
|-------|-----|---------------|--------------|
| 0 | `#ef4444` | `red-500` | strong red |
| 1 | `#22c55e` | `green-500` | strong green |
| 2 | `#3b82f6` | `blue-500` | strong blue |
| 3 | `#eab308` | `yellow-500` | gold |
| 4 | `#a855f7` | `purple-500` | violet |
| 5 | `#ec4899` | `pink-500` | magenta |
| 6 | `#14b8a6` | `teal-500` | dark teal |
| 7 | `#f97316` | `orange-500` | burnt orange |
| 8 | `#06b6d4` | `cyan-500` | cyan |
| 9 | `#84cc16` | `lime-500` | yellow-green |

The cyclic palette is a **presentation concern only** — it does not enter the
design token system (`index.css` / `tailwind.config.js`) because it is consumed
by raw `<canvas>` (dots) and raw `<svg>` (lines), not by Tailwind utility
classes. Listing it in the contract makes it auditable by the checker.

### Minutia dot colors (locked, reused from `useCanvasDrawer`)

Dots drawn on each canvas follow the existing `useCanvasDrawer` convention
(PRESERVED — `useCanvasDrawer` is unchanged per D-23, MATCH / D-24):

| Type | Color | Hex | Notes |
|------|-------|-----|-------|
| Termination (type 0) | red | `#ef4444` | matches PALETTE[0] but the value is hard-coded in `drawMinutiaDot` |
| Bifurcation (type 1) | green | `#22c55e` | matches PALETTE[1] but the value is hard-coded in `drawMinutiaDot` |
| Matched minutia dot | **PAIR color** | from `colorForIndex(pairIndex)` | overrides the type color when the minutia is part of a match pair |
| Highlighted (selected) | yellow ring | `#eab308` | `2.5px` stroke |
| Default border | white | `rgba(255,255,255,0.4)` | `1px` stroke |

A minutia that participates in a match pair is drawn with the **pair color**,
not the type color — this is the visual cue that tells the perito "this dot is
explained by a connecting line." The type still appears as a 1px inner dot in
the same termination/bifurcation color, inside the pair-colored ring.

### Destructive actions in Phase 23

There is **no destructive action in Phase 23** that requires confirmation:

- Enrollment is **forward-only**: once a capture is uploaded, the perito
  cannot delete it from the UI (deletion is an admin-side operation not in
  scope for this phase).
- Candidate selection is reversible (just click another candidate).
- Verdict submission is one-way, but the backend is idempotent on
  `case_id + verdict`; re-submitting the same verdict returns 200 silently
  and toasts "Decisión registrada" again. **No destructive confirm dialog
  needed.**

The "Cancelar" button on `/enroll` step 3 simply discards the in-memory
edited minutiae and returns to step 2 — it is non-destructive. **No `AlertDialog` is required in Phase 23.**

  dismissed with an X button in the top-right of the panel.
- The **verdict bar** stays at the bottom — same position as the current
  `ComparisonView` (lines 525-571).

### Match overlay details

- Each canvas is wrapped in a `<div className="aspect-square bg-black/95 ...">`
  to fix the display rect (RESEARCH §Pitfall 4 mitigation). Canvas internal
  resolution = `naturalWidth × naturalHeight`; CSS size = aspect-square box
  (max 480px square).
- The SVG layer uses `<svg className="absolute inset-0 w-full h-full pointer-events-none">` — pointer-events-none so it does not block future click handlers.
- Each canvas is captioned with a label `<p className="text-xs uppercase tracking-wider text-muted-foreground mb-2">` above the canvas.
- A stats badge `<Badge variant="secondary" className="font-mono text-[10px]">` overlays the gap between the two canvases (top center), showing "Pares matched: {n}" and "Similitud promedio: {pct}%".

### Touch targets

- All clickable cards / buttons follow the existing shadcn primitive heights:
  - `Button size="default"`: 40px tall
  - `Button size="sm"`: 36px tall
  - `Button size="icon"`: 40×40px
  - `CandidateCard` (custom): 64px min-height (4-row content: rank + name + document + score bar)
  - `MinutiaeEditor` mode buttons: 32px tall (`h-8`)
- Tap target minimum 40×40px is preserved everywhere (no sub-36px clickable surfaces).

### Responsive (best-effort, no mobile in MVP)

- Grid collapses to single column on `lg` breakpoint (`grid-cols-1 lg:grid-cols-2`).
- Match overlay canvases scale to 100% width, height auto.
- **Tablet/portrait** (768-1024px): two canvases stack vertically, SVG layer still spans the container.
- **Mobile** (<768px): out of scope (RESEARCH §Deferred — desktop perito workstation only).

## Error States

Errors are always surfaced through `useToast().addToast({ type: "error", title, description })`. The body is the `ApiError.message` (set by the `request<T>()` helper in `lib/api.ts`).

| Origin | Title | Description |
|--------|-------|-------------|
| 4xx file type | "Tipo de archivo inválido" | "Selecciona una imagen BMP, PNG o JPEG" |
| 413 file size | "Archivo demasiado grande" | "El archivo no debe exceder 10MB" |
| 4xx pipeline (no minutiae) | "No se detectaron minucias" | "La imagen no parece contener una huella dactilar válida." |
| 5xx search | "Error en la búsqueda" | "{error.message}" |
| 5xx enroll | "No se pudo procesar la imagen" | "{error.message}" |
| Network (fetch throws) | "Error de conexión" | "Verifica que el servidor backend esté en ejecución." |
| 404 case | "Caso no encontrado" | "El caso solicitado no existe o fue eliminado." |
| 404 person | "Persona no encontrada" | "La persona seleccionada ya no está registrada." |

For non-toast error surfaces (a Card with the error message, when a page-level
fetch fails), use the same pattern as `ComparisonView` (lines 337-356):

```tsx
<Card className="mt-6 border-destructive/50 bg-destructive/5">
  <CardContent className="flex items-center gap-3 py-6">
    <AlertCircle className="w-5 h-5 text-destructive shrink-0" />
    <p className="text-destructive font-medium">{message}</p>
  </CardContent>
</Card>
```

## Animation & Motion

- **Forbidden:** framer-motion, anime.js, or any animation library (RESEARCH
  §Deferred).
- **Allowed:** Tailwind's built-in `animate-in slide-in-from-*`, `fade-in-*`,
  `zoom-in-*` (already used by `toast.tsx` and `dropdown-menu.tsx`).
- **Allowed:** CSS transitions on color/opacity for hover states
  (`transition-colors`).
- **Allowed:** `animate-pulse` for skeleton loaders.
- **Match trace lines:** SVG `<line>` `stroke-opacity` transitions to 1.0
  on the first render of a new candidate (CSS transition 200ms) — implemented
  via `stroke-opacity: 0` initial + `useEffect` that flips to `similarity` after
  mount. This gives a "tracing" feel without an animation library.

## Accessibility (baseline)

- All clickable non-button elements (`CandidateCard`, `EmptyState` drop zone)
  have `role="button"`, `tabIndex={0}`, and `onKeyDown` for Enter/Space.
- All `<canvas>` elements have an `aria-label` describing the content
  (e.g., `aria-label="Huella latente con 12 minucias detectadas"`).
- The SVG line layer has `aria-hidden="true"` (it is decorative; the
  semantics live in the tabular trace below).
- Toast region is `role="status"` (set on the container in `toast.tsx`).
- Color contrast: all text on dark surfaces uses the `text-foreground` /
  `text-muted-foreground` tokens, which have ≥7:1 contrast against
  `bg-background` in dark mode (verified against WCAG AA).
- Match pair colors (#ef4444, #22c55e, etc.) are decoded in the
  **tabular trace** below the overlay (with a "Cilindro N" label per row).
  The colors are not the only channel; the perito can identify pairs by
  index even without color vision.
