<objective>
Implement the RAG Matching Engine (Weighted Mathematical Vector Search).

Purpose:
This is where the magic happens. A latent fragment query produces $M$ triangles (chunks). The database contains millions of chunks. For each query chunk, we find the $K$ closest database chunks using `pgvector` (`<->` or inner product). We then group the hits by their parent `fingerprint_id`, calculating the final score as: `Sum(Similarity * DB_Chunk_Weight)`. Fingerprints whose core matched perfectly will explode in score.

Output: High-performance, weight-aware SQL query and updated matching service.
</objective>
