<objective>
Implement the "Chunking & Weighting" core domain logic (The "Fingerprint RAG" approach). 

Purpose:
A fingerprint is no longer represented as a single global vector (which fails on partial latents). It is broken down into local invariant structures (triplets via Delaunay triangulation). Crucially, simulating the expert human heuristic, each triplet receives a weight (1.0 to 0.1) based on its spatial distance to the fingerprint's Core (Singularity). Triplets near the Core matter exponentially more than those at the noisy borders.

Output: Updated domain types and `RagTripletVectorizer`.
</objective>

Implement `RagTripletVectorizer` in `vectorizer.py`:
1. Extract minutiae points. If < 3, handle graceful fallback (e.g., pairs).
2. Apply `scipy.spatial.Delaunay` to create triangles.
3. For each triangle, calculate invariants:
   - 3 side lengths (normalized by max side to ensure scale invariance)
   - 3 internal angles
   - 3 minutiae types (encoded)
4. Retrieve the `Core` singularity from `ctx.singularities` (if detected by SingularityDetector).
5. Calculate the centroid of each triangle.
6. Compute `weight` based on distance from centroid to the Core. 
   - Strategy: Use an exponential decay function or Gaussian function `weight = exp(-distance^2 / (2 * sigma^2))` where `sigma` is a reasonable tuning parameter (e.g., 100 pixels). 
   - If no Core is found (blind latent), assign all weights = 1.0 (fallback strategy).
