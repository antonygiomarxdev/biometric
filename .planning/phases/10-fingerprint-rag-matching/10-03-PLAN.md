<objective>
Refactor the Database Schema and Repository pattern to support the 1-to-N RAG chunks.

Purpose:
To use Qdrant correctly with local structures, a single fingerprint must generate many rows in a chunk table (just like text embeddings). The vector column must have the correct dimension (3 sides + 3 angles + 3 types = 9-12 dims). We also need to store the `weight` associated with that vector chunk to perform weighted sum retrieval later.

Output: Alembic migration, updated SQLAlchemy models, and refactored Repository.
</objective>
