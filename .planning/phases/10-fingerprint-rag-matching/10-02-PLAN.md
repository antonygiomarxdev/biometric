<objective>
Implement Forensic Business Rules using the Strategy Pattern to govern the Fingerprint Service.

Purpose:
An expert discards latent fingerprints with fewer than 8 minutiae points if trying to establish a record, but will attempt to search a database with fragments (>= 2 points). We must model this explicitly. Applying Clean Architecture, these rules belong in the Domain layer, not deeply nested in HTTP routers or database repositories.

Output: Strategy pattern for forensic validation, integrated into `FingerprintService`.
</objective>
