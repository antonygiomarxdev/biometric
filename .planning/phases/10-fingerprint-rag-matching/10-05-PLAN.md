<objective>
Prove the RAG Dactilar system is correct via two complementary test strategies: property-based tests for mathematical invariants, and end-to-end tests for the full enroll→fragment-search flow.

Purpose:
The "smoke tests" written in Plan 10-04 (mocked FingerprintService, mock Session) verified the code wiring. They do NOT prove the algorithm is correct. For a forensic system, we need:

1. **Property-based tests (Hypothesis):** Generate arbitrary minutia configurations and assert the vectorizer's invariants hold for ALL of them, not just the 3-4 examples we hand-wrote.

2. **End-to-end tests with a real DB:** Enroll synthetic "persons" into a real (in-memory SQLite or testcontainer) PostgreSQL session, then search with a fragment and verify the correct person is returned.

3. **Invariants we must guarantee:**
   - **Indexing is stable:** enrolling the same minutiae twice produces the same chunk count.
   - **Rotation invariance:** rotating the query image by 0°/90°/180°/270° does not change the top match.
   - **Translation invariance:** cropping a fingerprint to any sub-window (>=2 minutiae) still finds the owner.
   - **Noise rejection:** a random fragment of 3+ spurious points should NOT rank highly against any enrolled person.

Output: Property-based test suite, E2E test suite, and updated documentation.
</objective>
