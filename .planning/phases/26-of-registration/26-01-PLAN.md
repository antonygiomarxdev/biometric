# Plan 26-01: OF Pre-Filter Pipeline

**Phase:** 26 (Orientation Field Registration)
**Goal:** Filter KNN candidates by global orientation field similarity
so that the triplet-based growing algorithm receives only candidates
whose global ridge structure is consistent with the probe. This is the
fix for Phase 25's 0/5 crop acceptance failure.
**Estimated tasks:** 7
**Files:** 4 new, 1 migration, 2 modified

## Architecture (recap)

```
Probe image
  â†’ enhance + thin + CN (existing)
  â†’ NEW: extract OF (OrientationFieldAnalyzer)
  â†’ KNN over triplet_features
  â†’ NEW: OF pre-filter (per-person RMS against enrolled OFs)
  â†’ grow_matches (existing)
  â†’ candidates
```

The filter is a **hard reject**: hits from `person_id` whose enrolled
OF has RMS > 0.50 against the probe OF are dropped before growing.

### T1: `of_similarity.py` â€” pure OF comparison

Create `apps/backend/src/processing/of_similarity.py` with:

- `OF_SIMILARITY_THRESHOLD: float = 0.50` (configurable via env)
- `OF_COHERENCE_MIN: float = 0.35` (matches `OrientationFieldAnalyzer` default)
- `class OFSimilarity` with:
  - `__init__(ori: np.ndarray, coh: np.ndarray) -> None` (stores the
    "reference" OF, e.g. the enrolled fingerprint)
  - `compare(other: "OFSimilarity") -> float` returns RMS in [0, ~1.4]
  - `compare_raw(other_ori, other_coh) -> float` for inline use
  - `@staticmethod build(image_or_enhanced: np.ndarray, block_size: int = 16) -> OFSimilarity`
    runs `OrientationFieldAnalyzer` on the input image
- `of_pseudo_core(coh: np.ndarray) -> tuple[int, int]` returns the
  coherence-weighted centroid as a (row, col) tuple. Used as the
  pseudo-anchor when `SingularityDetector` doesn't find a Core.
- `class OFSimilarityError(ValueError)` for invalid inputs

### T5: `mcc_matching_service.py` â€” wire OF filter into search

Modify `search_by_triplets` in `mcc_matching_service.py`:

- After `_run_quality_pipeline` returns, build probe OF
  via `OFSimilarity.build(result["enhanced_image"])`
- After KNN returns `all_hits`, **before** `grow_matches`:
  - Group hits by `person_id`
  - For each unique `person_id`, fetch its `OFRecord` via
    `OFRegistry.get_many(list_of_person_fingerprint_ids)` (caches)
  - Compute `OFSimilarity.compare()` between probe and each enrolled
  - If RMS > `OF_SIMILARITY_THRESHOLD` (0.50), **drop all hits from
    that person_id**
  - If all candidates are filtered out, return empty `candidates`
- Add a `dev_log("mcc.search.of_filter", ...)` line with
  `probe_persons=N`, `kept_persons=M`, `dropped_persons=N-M`,
  `min_rms=...`, `max_rms=...`

**Verification:**

## Goal-Backward Verification

The phase goal is **filter KNN candidates by OF so crop matches pass**.
This is verified by:

1. **T1-T3** build the building blocks. Verified by unit tests
   (`test_of_similarity.py`, `test_of_registry.py`).
2. **T4-T6** wire it into the matching pipeline. Verified by
   re-running `e2e_triplet_benchmark.py` (existing) and confirming
   the new behavior (5/5 self, â‰Ą4/5 crop50).
3. **T7** is the formal acceptance gate. Pass = the goal is met.

**Failure modes & rollback:**
- If crop match still fails: investigate per-person RMS values in
  `dev_log` output. Some persons may have noisy OFs (low coherence
  coverage). Consider lowering threshold to 0.40 (1st percentile
  in calibration).
- If latency exceeds 3s: pre-load all OFs in a single query at
  start of `search_by_triplets`, not per-hit.
- If self-match regresses: check that enrolled OFs are correctly
  persisted (T4 verification).
